# ADR-003: stdio transport now, SSE deferred

**Status:** Accepted

**Context:**
Claude Desktop uses stdio MCP transport. Tower uses SSE via mcp-proxy. Building for both simultaneously adds complexity with no immediate benefit.

**Decision:**
FastMCP stdio only for MVP. When absorbed into PrivyBot Phase 31, transport switches to SSE. The tool layer is transport-agnostic — only `server.py` changes.

**Consequences:**
Claude Desktop can use it immediately on Nitro. Tower deployment requires one server.py rewrite, no tool changes.
