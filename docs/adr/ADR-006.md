# ADR-006: Error handling and retry policy

**Status:** Accepted

**Context:**
External APIs fail. Network transient errors, rate limits, service outages all occur in production.

**Decision:**
Exponential backoff (2s, 4s, 8s) for transient errors (5xx, timeout, 429). Immediate failure for auth errors (401) and bad requests (400). Max 4 attempts for external APIs, 10 for SQLite WAL locks (inherited from DBManager). Failed publishes written to publish_log with error_message. No automatic retry of failed publishes — manual re-trigger only.

**Consequences:**
Transient failures recover automatically. Permanent failures surface clearly in get_publish_status. Dead letter is the publish_log table.
