# ADR-010: asyncio.run() pattern for async tests

**Status:** Accepted

**Context:**
Phase 3 attempted to use `pytest-asyncio` for async test execution. The plugin had loading issues across different Python environments and pytest versions. Debugging the plugin configuration consumed significant time without resolution. The async handlers themselves work correctly — the issue was purely test infrastructure.

**Decision:**
Remove `pytest-asyncio` from the project entirely. All async tests use `asyncio.run()` directly within synchronous test functions. This is the standard pattern going forward. No future async test should use `@pytest.mark.asyncio` decorators or any pytest-asyncio features.

**Pattern:**
```python
import asyncio

def test_async_handler(db):
    handler = AsyncHandler(db, "config")
    result = asyncio.run(handler.async_method("arg"))
    assert result is not None
```

**Consequences:**
- Tests are simpler and more portable across Python/pytest versions
- No plugin dependency or configuration issues
- `asyncio.run()` is stdlib — always available
- Test runtime includes actual backoff sleeps (2s, 4s, 8s) in retry tests — flag if runtime exceeds 30s by Phase 6
